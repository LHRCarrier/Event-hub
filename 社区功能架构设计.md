## 1. 需求分析

### 1.1 核心需求
| 需求编号 | 需求描述 | 优先级 |
| :--- | :--- | :--- |
| REQ-001 | 用户可创建社区，创建者自动获得管理员权限 | 高 |
| REQ-002 | 用户需先加入社区才能参与该社区活动 | 高 |
| REQ-003 | 社区管理员拥有社区管理全部权限 | 高 |
| REQ-004 | 普通成员可浏览社区、加入活动 | 高 |
| REQ-005 | 支持社区信息的增删改查 | 高 |
| REQ-006 | 支持用户加入/退出社区 | 高 |
| REQ-007 | 基于社区成员身份的活动权限控制 | 高 |

### 1.2 功能模块划分

## 2. 数据模型扩展方案

### 2.1 新增数据表

#### 2.1.1 communities 表（社区信息表）

| 字段名 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| community_id | INT | PRIMARY KEY, AUTO_INCREMENT | 社区ID |
| name | VARCHAR(100) | NOT NULL, UNIQUE | 社区名称 |
| description | TEXT | NULL | 社区描述 |
| logo_url | VARCHAR(255) | NULL | 社区Logo地址 |
| status | VARCHAR(20) | NOT NULL, DEFAULT 'ACTIVE' | 状态：ACTIVE/INACTIVE |
| create_time | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

| 字段名 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |

| community_id | INT | NOT NULL, FOREIGN KEY | 社区ID |
| user_id | INT | NOT NULL, FOREIGN KEY | 用户ID |
| role | VARCHAR(20) | NOT NULL, DEFAULT 'MEMBER' | 角色：ADMIN/MEMBER |
| status | VARCHAR(20) | NOT NULL, DEFAULT 'ACTIVE' | 状态：ACTIVE/INACTIVE |
| join_time | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 加入时间 |
| UNIQUE KEY | (community_id, user_id) | UNIQUE | 唯一约束 |

```java
@TableName("communities")
public class Community {
    @TableId(value = "community_id", type = IdType.AUTO)
    private Integer communityId;
    
    private String name;
    private String description;
    private String logoUrl;
    private String status;
    private Date createTime;
    private Date updateTime;
    
    // Getters and Setters
}
```

#### 3.1.2 Service接口 - CommunityService

| 方法名 | 功能描述 | 参数 | 返回值 |
| :--- | :--- | :--- | :--- |
| createCommunity | 创建社区（创建者自动成为管理员） | CommunityCreateRequest | CommunityResponse |
| updateCommunity | 更新社区信息（仅管理员） | communityId, CommunityUpdateRequest | void |
| getCommunityById | 获取社区详情 | communityId | CommunityResponse |
| getCommunities | 分页获取社区列表 | page, size, keyword | PageResponse |
| deleteCommunity | 删除社区（仅管理员） | communityId | void |
| getCommunityMembers | 获取社区成员列表 | communityId, page, size | PageResponse |

## 4. API接口规范

### 4.1 社区管理接口

#### 4.1.1 创建社区

| 属性 | 值 |
| :--- | :--- |
| **路径** | `/api/communities` |
| **方法** | `POST` |
| **描述** | 创建新社区，创建者自动成为管理员 |

**请求体：**
```json
{
  "name": "string (必填，社区名称)",
  "description": "string (选填，社区描述)",
  "logoUrl": "string (选填，Logo地址)"
}
```

**成功响应 (201)：**
```json
{
  "code": 201,
  "message": "创建成功",
  "data": {
    "communityId": 1,
    "name": "技术社区",
    "description": "技术爱好者交流社区",
    "logoUrl": null,
    "status": "ACTIVE",
    "createTime": "2024-05-12T10:00:00"
  }
}
```


## 7. 性能优化方案

### 7.1 数据库索引优化

| 表名 | 索引字段 | 索引类型 | 说明 |
| :--- | :--- | :--- | :--- |
| communities | name | UNIQUE | 社区名称唯一索引 |
| communities | status | NORMAL | 状态索引，加速状态筛选 |
| community_members | (community_id, user_id) | UNIQUE | 复合唯一索引 |
| community_members | user_id | NORMAL | 用户ID索引，加速用户社区查询 |
| community_members | community_id | NORMAL | 社区ID索引，加速成员列表查询 |
| events | community_id | NORMAL | 社区ID索引，加速社区活动查询 |

### 7.2 查询优化策略

| 优化项 | 策略 | 预期收益 |
| :--- | :--- | :--- |
| 社区成员查询 | 使用分页查询，限制单次返回数量 | 减少数据传输量 |
| 权限验证 | 缓存用户社区角色信息 | 减少数据库查询次数 |
| 社区列表 | 缓存热门社区列表 | 提升首页加载速度 |
| 活动列表 | 按社区ID分片查询 | 提升查询效率 |

### 7.3 缓存策略

| 缓存类型 | 缓存内容 | 过期时间 | 刷新时机 |
| :--- | :--- | :--- | :--- |
| 用户社区角色 | userId -> Map<communityId, role> | 5分钟 | 加入/退出社区时更新 |
| 社区基本信息 | communityId -> Community | 10分钟 | 社区信息更新时失效 |
| 社区成员计数 | communityId -> memberCount | 30分钟 | 成员变更时更新 |

## 10. 部署与集成方案

### 10.1 数据库迁移

```sql
-- 创建社区表
CREATE TABLE IF NOT EXISTS communities (
    community_id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL UNIQUE,
    description TEXT,
    logo_url VARCHAR(255),
    status VARCHAR(20) NOT NULL DEFAULT 'ACTIVE',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME ON UPDATE CURRENT_TIMESTAMP
);

-- 创建社区成员表
CREATE TABLE IF NOT EXISTS community_members (
    member_id INT PRIMARY KEY AUTO_INCREMENT,
    community_id INT NOT NULL,
    user_id INT NOT NULL,
    role VARCHAR(20) NOT NULL DEFAULT 'MEMBER',
    status VARCHAR(20) NOT NULL DEFAULT 'ACTIVE',
    join_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (community_id) REFERENCES communities(community_id) ON DELETE CASCADE,
    FOREIGN KEY (user_id) REFERENCES users(user_id) ON DELETE CASCADE,
    UNIQUE KEY unique_member (community_id, user_id)
);

-- 修改事件表添加社区字段
ALTER TABLE events ADD COLUMN community_id INT;
ALTER TABLE events ADD CONSTRAINT fk_events_community FOREIGN KEY (community_id) REFERENCES communities(community_id) ON DELETE SET NULL;
```
