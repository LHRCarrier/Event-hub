# EventHub 快速开始指南

## 一、项目简介

EventHub 是一个基于 Spring MVC 6 + MyBatis-Plus 的社区活动管理系统，提供活动管理、用户注册、活动报名等核心功能。

**技术栈**：
- 后端：Spring MVC 6.1.6 + MyBatis-Plus 3.5.5
- 前端：JSP + HTML + CSS + JavaScript + AJAX
- 数据库：MySQL 8.0
- 服务器：Apache Tomcat 10.x (Jakarta EE 9+)
- 构建工具：Maven 3.6+
- Java版本：JDK 17+

## 三、项目启动步骤

### 3.1 克隆项目

```bash
cd c:\Users\12321\Desktop
git clone <项目地址> Event-hub
cd Event-hub
```

### 3.2 配置数据库

#### 3.2.1 创建数据库

登录 MySQL 命令行客户端：

```bash
mysql -u root -p
```

执行以下命令创建数据库：

```sql
CREATE DATABASE IF NOT EXISTS eventhub CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE eventhub;
```

#### 3.2.2 初始化数据库表结构

在 MySQL 命令行中执行 `src/main/resources/sql/schema.sql` 文件：

```sql
SOURCE c:/Users/12321/Desktop/Event-hub/src/main/resources/sql/schema.sql;
```

或者使用以下命令在命令行执行：

```bash
mysql -u root -p eventhub < c:/Users/12321/Desktop/Event-hub/src/main/resources/sql/schema.sql
```

#### 3.2.3 验证初始化

执行以下命令查看表是否创建成功：

```sql
SHOW TABLES;
```

应显示以下表：
- users
- categories
- events
- registrations

#### 3.2.4 修改数据库配置

如果 MySQL 密码不是 `password`，请修改 `src/main/resources/application.properties` 文件：

```properties
jdbc.username=root
jdbc.password=您的MySQL密码
```

### 3.3 配置项目

#### 3.3.1 检查 JDK 版本

```bash
java -version
```

确保版本为 1.8 或更高。

#### 3.3.2 检查 Maven 版本

```bash
mvn -version
```

确保 Maven 版本为 3.6 或更高。

### 3.4 编译项目

在项目根目录下执行：

```bash
mvn clean compile
```

等待依赖下载完成，编译成功后会显示 `BUILD SUCCESS`。

### 3.5 打包项目

```bash
mvn clean package -DskipTests
```

打包成功后会生成 `target/eventhub.war` 文件。

### 3.6 部署到 Tomcat

#### 3.6.1 方式一：将 WAR 包复制到 Tomcat

将 `target/eventhub.war` 复制到 Tomcat 的 `webapps` 目录：

```bash
cp target/eventhub.war [TOMCAT_HOME]/webapps/
```

#### 3.6.2 方式二：使用 IDE 启动

在 IntelliJ IDEA 或 Eclipse 中：
1. 导入项目为 Maven 项目
2. 配置 Tomcat 服务器
3. 启动项目

### 3.7 访问应用

项目启动后，访问以下地址：

- 应用首页：http://localhost:8080/eventhub/
- API 基础路径：http://localhost:8080/eventhub/api/

## 六、常见问题

### 6.1 编译失败

**问题**：Maven 依赖下载失败

**解决方案**：
```bash
mvn clean install -U
```

### 6.2 数据库连接失败

**问题**：无法连接到 MySQL 数据库

**解决方案**：
1. 确认 MySQL 服务已启动
2. 检查 `application.properties` 中的数据库配置
3. 确认数据库用户权限

### 6.3 端口被占用

**问题**：8080 端口被其他应用占用

**解决方案**：
修改 `application.properties` 中的端口号：
```properties
server.port=8081
```

### 6.4 中文乱码

**问题**：返回的中文数据显示乱码

**解决方案**：
确认数据库连接字符串包含 `characterEncoding=utf8`：
```properties
jdbc.url=jdbc:mysql://localhost:3306/eventhub?useSSL=false&serverTimezone=UTC&allowPublicKeyRetrieval=true&characterEncoding=utf8
```
